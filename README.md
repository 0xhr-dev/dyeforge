# DyeForge

ブループロトコル:スターレゾナンス（BPSR / 星痕共鳴）の衣装染色用カラーマッチングツール。

参考画像から色をピックし、ゲームが表示する色と比較しながら、HEXコードを自動入力して目標色に収束させます。**人間の知覚に近いDelta E距離（業界標準）**で最適化するため、RGB数値上の近さではなく「見た目の近さ」で評価されます。

## 機能

- 任意のウィンドウからカラーピック（Target色の取得）
- リアルタイム監視（ゲーム画面の指定ピクセルを継続的にキャプチャ）
- **二種類の類似度表示**
  - RGB類似度（従来の%表示）
  - **Delta E（CIE Lab 知覚距離）**: 人間の目で見た近さを数値化
- 日本語のアドバイス表示（「ポインタを右上へ」など）
- HSV可視化（SV四角＋色相バーにTargetとCurrentの位置をマーカー表示）
  - **ゲーム到達不可能な領域**（彩度 > 78%）を暗く表示
  - Targetが範囲外の場合、到達可能位置への矢印も表示
- 自動調整（HEXコードを自動入力、適応型dampingと微調整モードで収束）
- ゲーム色範囲の自動対応（RGB 0x1A〜0xDA、HSV彩度上限78%）
- **到達可能HEX表示**: Targetが範囲外なら `#FF0000 → #DA4C4C` のように補正後の値を表示
- **👁 瞳モード**: 瞬きに強いサンプリング（6サンプル中央値で瞼色を自動排除）
- 最前面固定トグル

## Delta Eとは

Delta E (ΔE) は CIE Lab 色空間での2色間の距離で、**人間の視覚に近い色差評価**の業界標準です。

| ΔE | カテゴリ | 意味 |
|---|---|---|
| < 1 | 同一 | 人間の目では区別不可 |
| < 2 | ほぼ同一 | 訓練された目のみ判別可 |
| < 5 | 近い | 許容範囲 |
| < 10 | やや違う | 近い色だが差がある |
| < 20 | 違う | 明らかに違う色 |
| ≥ 20 | 全く違う | 全く違う色 |

RGB%だけでは分からない「見た目の近さ」を評価するため、**ゲーム限界で完全一致不可能な色でも最も視覚的に近い色**に自動収束します。

## 動作の流れ

1. Target色をピック（参考画像など任意のウィンドウから）
2. 監視ポイントを設定（ゲームのカラープレビュー位置）
3. キャリブレーション（ゲームのHEX入力欄を1回クリック）
4. 自動調整を実行
   - 初期: クランプ済みHEXを入力
   - ループ: 実測キャプチャ → 誤差計算 → 補正HEX送信
   - 振動検出時: 微調整モード（小さなステップで精度向上）
   - 最終: best値に復帰

## インストール

### ビルド済みEXEを使う（推奨）

[Releases](../../releases)ページから`DyeForge.exe`をダウンロード。

- Windows 10 / 11 64-bit
- 起動時にUACで管理者権限を要求（ゲームへの入力送信に必要）

### ソースから実行

```bash
pip install Pillow numpy pywin32
python color_matcher.py
```

## 使い方

| ボタン | 動作 |
|---|---|
| 色を取得 | ウィンドウ選択 → 画像から色をクリックで指定 |
| 監視設定 | BPSRウィンドウを自動検出 → 監視したい位置をクリック |
| calibrate | ゲームのHEX入力欄を1回クリックして位置を記憶 |
| 自動調整 | HEX入力＋補正ループを実行して目標色に収束 |
| pin | 最前面固定のON/OFF |

### 設定項目

- HEXのみ: カラーピッカーへのクリックをスキップし、HEX入力のみ（デフォルトON）
- **👁 瞳**: 自動調整時に6サンプル×150msの中央値で色を取得。瞬き（~0.3秒）を自動除外。瞳の色合わせに使用
- **ΔE閾値**: この値以下に達したら完了（デフォルト1.0、範囲0.5〜20）
- 回数: 補正ループの最大回数（デフォルト30回）

## 技術的な仕様

### ゲームの色制限
- **RGB範囲**: 各チャンネル `0x1A`〜`0xDA` (26〜218) にクランプ
- **彩度上限**: HSV彩度 ≤ 78%（全色相・明度でフラット、120点テストで確認済み）
- **ガンマ変換**: BPSRはHEX入力にsRGB風のガンマをかける（例: 入力`0x80`→出力約196）

### 補正アルゴリズム
- **適応型damping**: 初期0.5 → 終盤0.95、誤差サイズに応じた調整
- **振動検出**: 過去3回の実測と比較、繰り返しパターン検出
- **微調整モード**: 振動検出後、damping 0.15〜0.25で精密調整
- **best追跡**: ΔE最小値を常に記録、終了時に最良値に自動復帰

### 瞳モードのサンプリング
通常モード: 2サンプル×150ms の平均
瞳モード: 6サンプル×150ms の**各チャンネル中央値**
- 瞬き(~0.3秒)は最大2サンプルを汚染 → 中央値で自動除外
- 全6サンプルが瞬き中になる確率は10^-6 (実質無視可能)

### 入力方式
Win32 `SendInput` APIでキーボード/マウス入力をシミュレート。キャプチャは `PrintWindow` と `BitBlt` のフォールバック。

## ログ

実行ログは `%USERPROFILE%\dyeforge.log` に出力されます（補正ループの各ステップを記録、デバッグ用）。

## 制限事項

- ゲームが管理者権限で動作している場合、DyeForge側も管理者権限が必要
- キャリブレーション情報は終了時に破棄されるため、起動のたびにキャリブレーションが必要
- HEX入力欄が存在するゲームが対象（BPSRで動作確認）
- 自動調整の精度はゲームの色空間に依存
  - 範囲内の色（彩度70%以下）: ΔE 1〜3 (ほぼ同一)
  - 境界付近の色（彩度75%前後）: ΔE 2〜5 (近い)
  - 範囲外の色（純赤・純青等）: ΔE 15〜30 (ゲーム上限)
